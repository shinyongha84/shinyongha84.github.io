ㆍKotlin(Android)

MeatWatch, MTrace 등 축산물 관련 API 를 이용해 축산물의 정보를 가져와 기록하게 개발하였습니다.

Project Review

ㆍ아쉬운 부분
MVVM 패턴을 완벽히 살리지 못했습니다.
MVVM 패턴을 처음 사용해본 프로젝트였는데 결국 이 후에 MVVM 중 View 에 데이터가 몰리는 현상이 발생했습니다.
View Model 에 데이터를 보관하고, XML 에서는 View Model 만 참조하고, View 에서는 View Model 에 있는 데이터를 꺼내 썼어야 했는데 작성하다보니 View 에 데이터가 몰리게 되었습니다.

ㆍ전체적인 Review
Android 의 기반을 다져주는 프로젝트 였다고 생각합니다.
이 프로젝트로 인해 Android Thread 구조나, Intent 에 대한 활용법, Data Binding 등 많은 배움을 얻었습니다.
하지만 위에 기술된 바와 같이 MVVM 패턴을 제대로 살리지 못해 결국 코드가 스파게티 코드가 되어 아쉬움이 남는 프로젝트입니다.
계속해서 수정을 거치고 있으나, 처음 기반이 잘못 잡혀 수정을 하려면 모든 프로젝트를 들어내야 할 것으로 보입니다.
React 에 대해서도 접근을 하였으나 함수형 컴포넌트가 아닌 클래스형 컴포넌트로 접근을 했고, 이 후에는 PDA 작업으로 인해 React 쪽의 전반적인 내용을 다른 팀원이 작성했었습니다.
이 때 같이 React 에 대해 공부하며 경험했다면, 더 좋은 프로젝트 경험이 되지 않았을까 합니다.
