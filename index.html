Interested

최근 TDD에 대해 관심을 갖고 있습니다.
Front-End에 적용할 경우 Layout에 대해 결과값이 잘 나오는지 직접 결과물을 보지 않고 검증을 할 수 있는게 매력적이게 느껴졌습니다.
그 외에 TDD를 적용하게 될 경우 Process/Logic 을 수정 하였을 때 테스트 케이스가 잘 작성이 되어 있다면 FullTest 를 진행하지 않고, TDD를 통해 검증할 수 있다는 것이 상당히 매력적입니다.
TDD를 자세히 공부하여 초기엔 힘들겠지만 테스트 케이스를 잘 작성하고 개발을 진행한다면 작업 효율이 크게 상승할 것으로 기대하고 있습니다.
하지만, 당장 TDD를 작성하라 하면 못할 것 같아 일단 BDD의 개념으로 진행을 해보려고 하고 있습니다.
공부한 내용을 개인 프로젝트에 적용할 예정입니다.

Spring Boot를 통한 Back-end 코딩에도 흥미가 생겼고,
(Front-end 개발자들이 원하는 데이터를 최소화 하여 보내주거나, 핵심 로직을 작성해 프로그램이 잘 돌아가는 것을 보면 참 재밌습니다.)
Front-end 에서 React.js Library 를 공부하며 공부한 내용을 썼을 때에도 성취감이 있어 React.js를 주로 공부중입니다.

What I'm studying

최근 TDD 와 GraphQL 을 배우고 있습니다.

TDD 를 공부하게 된 계기는 테스트 주도 개발의 중요성을 프로젝트를 진행하며 많이 느꼈기 때문입니다.
TDD의 내부 개념인 BDD를 통해 테스트 케이스를 수월하게 작성하기 위해 노력하고 있습니다.

GraphQL은 FE 개발을 진행하며 원하는 데이터를 불러오기 위해 Over-fetching을 할 때가 많아 이에 대한 방안으로 GraphQL을 선택하였습니다.
이 뿐만 아니라 Back-end를 너무 깊이 의존하고 있어 응답에 필드를 추가하거나, 삭제, 신규생성할 때 항상 BE 개발자에게 요청을 드렸는데,
이는 너무 효율적이지 않은 것 같았습니다.
실제 업무에 적용하기 위해 apollo-client를 이용해 여러 시도를 해보고 있습니다.

공부한 내용에 대해서는 모두 기술 블로그에 작성하고 있습니다.
일주일에 2~3개의 내용에 대해 작성하고자 노력하고 있습니다.
